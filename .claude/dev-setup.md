# Dev setup

Everything runs inside `devenv shell` so local == CI. The devenv scripts are the
same ones the GitHub Actions workflow calls — use them, don't reinvent.

## Prerequisites

- [devenv](https://devenv.sh) + Nix. Enter the environment with `devenv shell`
  (or use direnv). Node, npm, and the Playwright browsers all come from Nix —
  never `npm install` Node or `npx playwright install`.
- All commands below assume you're **inside** `devenv shell`.

## Everyday commands

| Command     | What it does                                                |
|-------------|-------------------------------------------------------------|
| `install`   | `npm ci` — clean install from the lockfile.                 |
| `dev`       | Docusaurus dev server on **port 3000** (builds uikit first). |
| `build`     | Full production build: uikit, then the website.             |
| `lint`      | ESLint across the whole repo, `--max-warnings 0`.           |

`npm run typecheck` runs `tsc` per workspace if you want types checked separately.

## Layout

npm workspaces:

- `uikit` — antd-based UI kit. The website depends on it, so it builds first
  (`prestart` / `build:uikit`). Change uikit → restart `dev`.
- `website` — the Docusaurus site (marketing pages + docs).

Docs content lives in `website/docs/`. The sidebar is **explicit** in
`website/sidebars.ts` — add a new doc there, it is not autogenerated.

## Before you push

1. `lint` — must be clean (zero warnings).
2. `build` — must succeed. CI runs the exact same script on linux-x64.

If you touched dependencies, read the lockfile / native-bindings section in the
root `CLAUDE.md` before committing — regenerating the lockfile the wrong way
breaks CI silently.
